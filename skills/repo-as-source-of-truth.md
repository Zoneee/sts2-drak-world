# Skill：仓库即唯一事实来源

## 目的
确保持久性项目知识存放在仓库内并可被发现。

## 规则
1. 不依赖对话记录、内存或私下讨论作为唯一信息来源。
2. 如果某条规则、决策或模式需要反复使用，将其记录在仓库文档中。
3. 优先使用版本化的 Markdown、Schema、计划和自动生成的参考文档，而非临时说明。
4. 若仓库文档与代码冲突，应调查并协调两者。
5. 若代码改动使文档失效，必须在同一次改动中更新文档。

## 阅读顺序
1. `AGENTS.md`
2. `docs/index.md`
3. 相关产品规格
4. 相关架构文档
5. 相关开发规范
6. 相关当前计划
7. 必要时查阅自动生成的参考文档

## 完成要求
若工作中产生了新的持久性知识，必须将其写入仓库。
