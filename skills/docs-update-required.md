# Skill：文档必须同步更新

## 目的
保持仓库知识与代码改动同步。

## 需要更新文档的情况
- 行为发生变化
- API 发生变化
- 架构发生变化
- 操作步骤发生变化
- Feature Flag 或配置发生变化
- 质量预期发生变化

## 候选更新位置
- `docs/product-specs/`
- `docs/architecture/`
- `docs/standards/`
- `docs/runbooks/`
- `docs/exec-plans/`
- `docs/generated/`

## 规则
若未来的贡献者或 Agent 需要了解这些信息，就必须将其写下来。
