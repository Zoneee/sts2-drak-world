# Skill：文档必须同步更新

## 目的
保持仓库知识与代码改动同步。

## 需要更新文档的情况
- 行为发生变化
- API 发生变化
- 架构发生变化
- 操作步骤发生变化
- Feature Flag 或配置发生变化
- 质量预期发生变化

## 候选更新位置
- `docs/product-specs/`
- `docs/architecture/`
- `docs/standards/`
- `docs/runbooks/`
- `docs/exec-plans/`
- `docs/generated/`

## 规则
若未来的贡献者或 Agent 需要了解这些信息，就必须将其写下来。

## 执行计划的存档
- 已完成的执行计划必须从 `docs/exec-plans/active/` 移动到 `docs/exec-plans/completed/`。
- 归档后的计划保留供后续贡献者和 Agent 回溯决策、证据与后续事项。
