# Skill：边界校验

## 目的
在边界处校验所有不可信输入，保护系统安全。

## 适用范围
- 请求载荷（Payload）
- 查询参数
- 环境变量
- 外部 API
- Webhook
- 形状不确定的数据库读取结果
- LocalStorage / 浏览器存储
- 文件上传
- 第三方集成

## 规则
1. 在边界处解析并校验。
2. 不向业务逻辑传递形状未知的数据。
3. 返回明确的校验错误。
4. 校验后优先使用强类型的内部表示。
5. 为合法和非法情况添加或更新测试。
6. 必要时在 `docs/standards/api-contracts.md` 中记录新的接口约定。

## 最低证据要求
- 正常路径的测试
- 非法形状 / 格式错误输入的测试
- 注明校验发生的位置
