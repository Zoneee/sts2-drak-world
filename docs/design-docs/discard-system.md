# 弃牌体系设计文档

状态：有效
负责人：产品与工程团队
最后评审：2026-03-26

## 设计目标
这套卡组的核心体验不是“打出才有收益”，而是让同一张牌在两个时点都具备价值：
- 打出时提供铺垫、整理或保底
- 被弃掉时提供真正的体系兑现

围绕这个目标，当前体系追求三件事：
- 玩家能够主动制造弃牌
- 被弃掉的牌本身值得围绕它构筑或留作链路节点
- 攻击、防御、减益和手牌整理都通过弃牌链路形成层级分工

## 当前已落地内容
### 核心牌组

### 诊断与可观察性
为了让设计验证可落地，当前实现把以下信号做成了常规日志：
- 模组初始化
- 卡牌自动发现
- `ModelDb` 进库与进池
- 弃牌命令链路
- 单卡 `OnPlay()` 与弃牌触发
- 弃牌来源与本回合弃牌次数

## 当前规则取舍
当前版本采用收紧后的规则：

> 默认只有因卡牌效果导致的弃牌，才会触发这张牌的弃牌效果。

唯一显式例外：

> `FadingFormula` 在牌面明确写明，回合结束自弃也会触发其弃牌效果。

打出约束：

> 带有打出时弃手牌要求的卡，只有在当前手中已经有足够其他手牌可弃时才能打出；若存在待生效的额外弃牌次数，也一并计入。

优点：
- 主动弃牌重新成为体系核心动作
- 可以放心提高弃牌收益，不必担心所有被动弃牌都白拿回报
- 稀有规则例外牌能更清晰地承担惊喜体验

代价：
- 需要运行时记录“因卡牌效果弃牌”的上下文
- `FadingFormula` 这类显式例外牌必须在牌面说清楚
- 链路型弃牌收益会更依赖中继牌数量与手牌管理

## 当前设计收益
- 每张牌同时有打出价值与弃牌价值，减少“抽到但不想打”的死牌感。
- 体系可以通过抽弃链条把局部收益叠成一个更大的回合。
- 费用、稀有度与身份已经拉开：common 负责启动，uncommon 负责转换，rare 负责引擎、终结与规则例外。

## 高风险设计点
### `ShatteredEcho`
高压缩引擎在升级后会继续放大下一次弃牌链路，需要重点观察是否造成异常高频弃牌回合。

### `FinalDraft`
作为稀有终结件，阈值奖励若过高，可能让第二次弃牌后的战斗 swing 过早失衡。

### `FadingFormula`
作为明确规则例外牌，需要重点验证奇偶弃牌判定是否清晰、回报是否足够戏剧化且不过稳。

## 后续方向
- 给“因卡牌效果弃牌”引入正式关键词或规则词。
- 增加与弃牌联动的能量牌、返手牌或防御终端牌。
- 评估能力牌、消散牌和“打出两次后消散”类高辨识度设计。
- 继续完善卡图资源导出与部署流程，减少资源问题对设计验证的干扰。
- 做实机数值验证，优先覆盖 `ShatteredEcho`、`FinalDraft`、`FadingFormula`。
