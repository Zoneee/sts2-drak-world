# 弃牌体系设计文档

状态：有效
负责人：产品与工程团队
最后评审：2026-03-24

## 设计目标
这套卡组的核心体验不是“打出才有收益”，而是让同一张牌在两个时点都具备价值：
- 打出时提供基础节奏
- 被弃掉时提供二次收益

围绕这个目标，当前体系追求三件事：
- 玩家能够主动制造弃牌
- 被弃掉的牌本身值得保留在手里或送入循环
- 攻击、防御、过牌和减益都能通过弃牌链路获得回报

## 当前规则取舍
当前版本采用最直接的规则：

> 只要这张牌被弃掉，就触发它的弃牌效果。

优点：
- 实现简单
- 日志验证直接
- 玩家规则学习成本低

代价：
- 还没有区分“主动弃牌”和“回合结束弃牌”
- 某些带自动弃牌能力的牌更容易越过触发门槛
- 后续平衡时可能需要重新定义触发边界

## 当前设计收益
- 每张牌同时有打出价值与弃牌价值，减少“抽到但不想打”的死牌感。
- 体系可以通过抽弃链条把局部收益叠成一个更大的回合。
- 攻击、防御、减益和过牌都能嵌入弃牌机制，构筑方向更完整。

## 高风险设计点
### `ShatteredEcho`
高抽牌回转可能形成过强的引擎回环。

### `FinalDraft`
作为稀有牌，单卡 swing potential 较高，需要重点看是否过早决定战斗。

### `FadingFormula`
由于会在回合结束自弃，天然绕开了“必须主动弃牌”的门槛，需要重点验证是否过稳。

## 后续方向
- 给“弃牌触发”引入正式关键词或规则词。
- 评估是否只允许“因卡牌效果弃牌”时触发。
- 增加与弃牌联动的能量牌、返手牌或防御终端牌。
- 继续完善卡图资源导出与部署流程，减少资源问题对设计验证的干扰。
- 做实机数值验证，优先覆盖 `ShatteredEcho`、`FinalDraft`、`FadingFormula`。
