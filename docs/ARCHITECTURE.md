## Core Components

### 1. **Main.cs** — Mod Loader

**Responsibility**: Register mod with BaseLib-StS2, initialize card system

```csharp
// Pseudo-code structure
public class DiscardModMain : BaseMod {
    public override void RegisterCards() {
        // Register all 8 cards via CardRegistry
        // BaseLib automatically adds to card pool
    }
    
    public override void OnGameStart() {
        // Hook into discard pile events
        // Initialize logging
    }
}
```

**Key Events**:
- `OnModLoad()` — Called when game initializes mod
- `RegisterCards()` — Called when populating card pool
- `OnGameStart()` — Called when run begins

---

### 2. **Cards/IDiscardTrigger.cs** — Interface

**Responsibility**: Define contract for discard-trigger cards

```csharp
public interface IDiscardTrigger {
    bool OnDiscard(AbstractCard card, AbstractPlayer player);
    // Returns: true if effect executed successfully
}
```

**All discard-trigger cards implement this interface** to enable:
- Centralized discard event detection
- Type-safe card filtering (e.g., in SwiftCut, check if discarded card is IDiscardTrigger)
- Extensibility for new card types

---

### 3. **Cards/[CardName].cs** — Individual Cards

**Example: DarkFlameFragment.cs**

```csharp
public class DarkFlameFragment : CustomCard, IDiscardTrigger {
    public override void Initialize() {
        // Set ID, name, cost, rarity, etc.
        // BaseLib handles stat initialization
    }
    
    public override bool CanPlay() => false;  // Cannot play from hand
    
    public bool OnDiscard(AbstractCard card, AbstractPlayer player) {
        // Deal 6 damage to all enemies
        // Log: [DiscardMod] DarkFlameFragment discarded
        return true;
    }
}
```

**Card Template** (Copy for new cards):
1. Inherit from `CustomCard` (BaseLib)
2. Implement `IDiscardTrigger` interface
3. Override `Initialize()` for card metadata
4. Override `CanPlay()` to return false (if no-play card)
5. Implement `OnDiscard()` for trigger effect
6. Register in `Main.RegisterCards()`

### 4. **Patches/DiscardHooks.cs** — Discard Event Detection

**Responsibility**: Hook into discard pile events (Fallback if BaseLib doesn't expose events)

**Strategy**: Use Harmony 2.x to patch `AbstractCard.deactivate()` or `AbstractPlayer.discardCard()`

```csharp
// Pseudo-code
[HarmonyPatch(typeof(AbstractPlayer), nameof(AbstractPlayer.discardCard))]
public static class DiscardPatch {
    public static void Postfix(AbstractCard card) {
        if (card is IDiscardTrigger trigger) {
            trigger.OnDiscard(card, AbstractDungeon.player);
        }
    }
}
```

**When to Use**:
- BaseLib `CustomCardModel` doesn't expose discard events
- Need to intercept discard pile updates
- Emergency workaround for breaking changes

**References**:
- Harmony docs: https://harmony.pardeike.net/
- emptylower LAN mod: For discard patch examples

### 5. **Utils/Logger.cs** — Standardized Logging

**Responsibility**: Centralized debug output with `[DiscardMod]` prefix

```csharp
public static class Logger {
    public static void Log(string message) {
        Debug.Log($"[DiscardMod] {message}");
    }
    
    public static void LogWarning(string message) {
        Debug.LogWarning($"[DiscardMod] WARNING: {message}");
    }
    
    public static void LogError(string message) {
        Debug.LogError($"[DiscardMod] ERROR: {message}");
    }
}

// Usage
Logger.Log("DarkFlameFragment discarded, dealing 6 damage");
```

**Why Prefix?** Makes mod logs easily searchable in STS2 console via `grep` or text filtering.

---

### 6. **Utils/CardRegistry.cs** — Centralized Card Management

**Responsibility**: Single source of truth for card IDs and lookups

```csharp
public static class CardRegistry {
    public const string DARK_FLAME_ID = "DiscardMod_DarkFlameFragment";
    public const string TOXIN_RECORD_ID = "DiscardMod_ToxinRecord";
    // ... etc
    
    public static List<Type> GetAllCards() {
        return new List<Type> {
            typeof(DarkFlameFragment),
            typeof(ToxinRecord),
            // ...
        };
    }
    
    public static bool IsDiscardTrigger(string cardID) {
        return cardID.StartsWith("DiscardMod_");
    }
}
```

**Why Centralize?** Prevents ID duplication bugs, enables bulk operations (e.g., "disable all discard-trigger cards").

## Discard Detection: Two Approaches

### **Approach A: BaseLib Hooks (Preferred)**

**How it works**:
1. BaseLib exposes `CustomCardModel.OnDiscard()` event
2. All IDiscardTrigger cards subscribe to event
3. Automatic registration in card registry

**Pros**: Clean, maintainable, auto-synced with game updates  
**Cons**: Requires BaseLib 1.8.0+; limited if BaseLib doesn't expose hooks

**Implementation** (Pseudo-code):
```csharp
public class Main : BaseMod {
    public override void RegisterCards() {
        foreach (var cardType in CardRegistry.GetAllCards()) {
            var card = Activator.CreateInstance(cardType) as CustomCard;
            if (card is IDiscardTrigger trigger) {
                card.OnDiscardEvent += (c, p) => trigger.OnDiscard(c, p);
            }
        }
    }
}
```

### **Approach B: Harmony Patches (Fallback)**

**How it works**:
1. Patch `AbstractPlayer.discardCard()` using Harmony
2. Check if discarded card implements IDiscardTrigger
3. Manually invoke trigger logic

**Pros**: Works even if BaseLib doesn't expose hooks  
**Cons**: Requires runtime patching; fragile if STS2 code changes

**Implementation** (Pseudo-code):
```csharp
[HarmonyPatch(typeof(AbstractPlayer))]
public static class DiscardPatch {
    [HarmonyPostfix]
    [HarmonyPatch(nameof(AbstractPlayer.discardCard))]
    public static void PostDiscardCard(AbstractCard card) {
        if (card is IDiscardTrigger trigger) {
            trigger.OnDiscard(card, AbstractDungeon.player);
        }
    }
}
```

**Decision Point (Phase 2)**:
- Research BaseLib v0.99+ docs for discard event API
- If exists: Use Approach A
- If doesn't: Fall back to Approach B with DiscardHooks.cs

## Multiplayer Sync

### **Mod Version Pinning**

STS2 enforces mod version matching for multiplayer games:
- All players must have identical mod `.dll` file (binary match)
- Game refuses connection if mods differ
- No custom sync code needed; game handles this natively

### **No Network Communication**

Discard-trigger effects are **deterministic** and fire on each player's client independently:
- Player A discards → fires locally → sends to other players as normal game action
- STS2 handles sync via existing action replication system
- Mod doesn't need to implement custom networking

### **Testing Approach**

1. **Host**: Create multiplayer game
2. **Client**: Connect (both must have mod v0.1.0-alpha)
3. **Play**: Discard-trigger cards → verify effects appear correctly on both clients
4. **Verify**: Check console logs on both clients for `[DiscardMod]` prefix

## Troubleshooting

### **Mod not loading in-game**

- Check console for `[DiscardMod] Loaded` message
- Verify `.dll` in correct mods folder (see README.md)
- Check [DEBUGGING.md](DEBUGGING.md) for detailed steps

### **Discard effects not firing**

- Verify card is discarded (not played) via console logs
- Check if `IDiscardTrigger.OnDiscard()` is registered properly
- Investigate Harmony patch vs. BaseLib hook (see "Discard Detection" above)

### **Multiplayer desync**

- Ensure all players running identical mod version
- Check for mod conflicts with RMP, Advisor (common mods)
- Report detailed logs to GitHub Issues
