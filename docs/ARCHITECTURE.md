## Core Components

### 1. **Main.cs** — Mod Entry Point

**Responsibility**: Initialize mod, register cards into the game's card pool.

```csharp
[ModInitializer(nameof(Initialize))]
public static class DiscardModMain
{
    public const string ModId = "STS2DiscardMod";
    public static Logger Logger { get; } = new(ModId, LogType.Generic);

    public static void Initialize()
    {
        Logger.Info("loading...");
        RegisterCards();
        Logger.Info("loaded!");
    }

    private static void RegisterCards()
    {
        ModHelper.AddModelToPool(typeof(RegentCardPool), typeof(DarkFlameFragment));
        ModHelper.AddModelToPool(typeof(RegentCardPool), typeof(SwiftCut));
        ModHelper.AddModelToPool(typeof(RegentCardPool), typeof(ToxinRecord));
        ModHelper.AddModelToPool(typeof(RegentCardPool), typeof(ShatteredEcho));
        Logger.Info("Registered 4 discard-trigger cards to RegentCardPool");
    }
}
```

**Key namespaces**:
- `MegaCrit.Sts2.Core.Logging` — `Logger`, `LogType`
- `MegaCrit.Sts2.Core.Modding` — `ModInitializer`, `ModHelper`
- `MegaCrit.Sts2.Core.Models.CardPools` — `RegentCardPool`

### 2. **Cards/[CardName].cs** — Individual Cards

All cards extend `CardModel` from `MegaCrit.Sts2.Core.Models`.

**Template:**

```csharp
using MegaCrit.Sts2.Core.Entities.Cards;
using MegaCrit.Sts2.Core.GameActions.Multiplayer;
using MegaCrit.Sts2.Core.Models;
using System.Threading.Tasks;

namespace DiscardMod.Cards;

public class MyCard : CardModel
{
    public MyCard()
        : base(cost: 1, CardType.Skill, CardRarity.Common, TargetType.Self, showInLibrary: true) { }

    public override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
    {
        // TODO: implement play effect
        DiscardModMain.Logger.Info("MyCard played");
        await Task.CompletedTask;
    }

    public override void OnUpgrade()
    {
        // TODO: implement upgrade effect
    }
}
```

**CardModel constructor parameters**:

| Parameter       | Type         | Notes                                                       |
| --------------- | ------------ | ----------------------------------------------------------- |
| `cost`          | `int`        | Energy cost (0–5)                                           |
| `CardType`      | `CardType`   | `Attack`, `Skill`, `Power`, `Status`, `Curse`               |
| `CardRarity`    | `CardRarity` | `Starter`, `Common`, `Uncommon`, `Rare`, `Special`, `Curse` |
| `TargetType`    | `TargetType` | `AnyEnemy`, `AllEnemies`, `Self`, `None`                    |
| `showInLibrary` | `bool`       | Show in card library (true for normal cards)                |

**Localization**: Card titles and descriptions live in `src/localization/eng/cards.json`. The key format is `CLASSNAME.title` / `CLASSNAME.description` (uppercase class name).

## Build & Deployment

### Local Build

```bash
cd src/
dotnet build --configuration Release
# Output: src/bin/Release/net9.0/STS2_Discard_Mod.dll
```

### Auto-Deploy (MSBuild)

The csproj includes a `CopyToModsFolderOnBuild` target that automatically copies the DLL and `STS2_Discard_Mod.json` to a subfolder in the game's mods directory after every successful build, **if the mods directory exists**.

Default mods paths:
- **Linux**: `~/.local/share/Steam/steamapps/common/Slay the Spire 2/mods/`
- **Windows**: `D:\G_games\steam\steamapps\common\Slay the Spire 2\mods\`
- **macOS**: `~/Library/Application Support/Steam/steamapps/common/Slay the Spire 2/SlayTheSpire2.app/Contents/MacOS/mods/`

Override the path at build time:
```bash
dotnet build src/ --configuration Release -p:SteamLibraryPath=/custom/path/steamapps
```

The deploy target creates a **subfolder** named `STS2_Discard_Mod/` inside the mods folder:
```
{game}/mods/
└── STS2_Discard_Mod/
    ├── STS2_Discard_Mod.dll
    └── STS2_Discard_Mod.json
```

### CI/CD (GitHub Actions)

- **Trigger**: Push to `main` or tag `v*`
- **Requires**: `STS2_DLL_B64` repository secret (base64-encoded `sts2.dll`)
- **Artifacts**: `dist/STS2_Discard_Mod/` zip uploaded to GitHub Actions
- **Releases**: On tag push, `softprops/action-gh-release` creates a GitHub Release

```bash
git tag v0.1.0-alpha && git push origin v0.1.0-alpha
# GitHub Actions auto-creates Release + zip artifact
```

## Logging

The mod uses `MegaCrit.Sts2.Core.Logging.Logger` directly (no custom wrapper).

```csharp
// Declaration in Main.cs
public static Logger Logger { get; } = new(ModId, LogType.Generic);

// Usage in cards or other classes
DiscardModMain.Logger.Info("message");
DiscardModMain.Logger.Debug("debug info");
DiscardModMain.Logger.Error("something failed");
```

Log lines appear in BepInEx console/log prefixed with `[STS2DiscardMod][INFO]`.

BepInEx log file: `{game_dir}/BepInEx/LogOutput.log`

## Discard Trigger Mechanic (Planned)

The core "弃触发" mechanic is **not yet implemented**. The planned approach:

1. Use Harmony to patch the game's discard method (e.g., `AbstractPlayer.discardCard` or equivalent in STS2)
2. When a card is discarded, check if it has a pending trigger effect
3. Execute the trigger effect via the game's action queue

Implementation is blocked pending research into the STS2 API for discard events. See `docs/DEV_GUIDE.md` for the next steps.

## Troubleshooting

**Mod not loading**: Check `{game}/BepInEx/LogOutput.log` for `[STS2DiscardMod]`. See [DEBUGGING.md](DEBUGGING.md).

**Card not appearing in pool**: Verify `ModHelper.AddModelToPool(typeof(RegentCardPool), ...)` is called and the localization JSON has the card's keys.

**Build error STS001**: Missing localization key — add it to `src/localization/eng/cards.json`.
