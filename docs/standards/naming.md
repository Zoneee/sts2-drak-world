# 命名规范

状态：有效
负责人：工程团队
最后评审：（填写日期）

## 通用原则
- 名称应清晰描述用途
- 避免缩写（除非是广泛接受的约定）
- 保持一致性优先于"更好的名字"

## 文件命名
- 使用小写连字符（kebab-case）
- 文件名应反映其主要导出内容

## 变量与函数
- 变量：camelCase
- 常量：UPPER_SNAKE_CASE
- 函数：camelCase，动词开头

## 类与类型
- PascalCase

## 数据库
- 表名：小写蛇形（snake_case），复数
- 列名：小写蛇形（snake_case）
