# 分层规则

状态：有效
负责人：工程团队
最后评审：（填写日期）

## 标准分层顺序
类型层 → 配置层 → 数据层 → 服务层 → 运行时层 → UI 层

## 各层说明
- **类型层**：数据定义与 Schema 对齐的表示
- **配置层**：配置、Feature Flag、环境变量访问
- **数据层**：持久化与外部 IO 访问
- **服务层**：业务规则与用例
- **运行时层**：编排、启动、执行连接
- **UI 层**：展示与交互层

## 规则
- 一层只能依赖允许的下层/前层。
- UI 层不得直接访问数据层。
- 服务层不得依赖 UI 层。
- 数据层不得导入 UI 层的关注点。
- 横切关注点应通过显式 Provider 接口引入。

## 强制执行方式
- 结构测试
- 自定义 Lint 规则
- PR 评审清单
