# 系统不变量

状态：草稿
负责人：工程团队
最后评审：（填写日期）

## 用途
记录系统中必须始终成立的约束，任何改动不得违反这些不变量。

## 格式
每条不变量包含：
- 编号
- 描述
- 适用范围
- 验证方式

## 不变量列表

| 编号 | 描述 | 适用范围 | 验证方式 |
|------|------|----------|----------|
| INV-001 | 用户认证状态必须在服务层验证，不得仅依赖 UI 层 | 认证模块 | 集成测试 |
| INV-002 | 外部输入在进入业务逻辑前必须经过校验 | 所有边界 | Lint 规则 + 测试 |

（根据实际项目补充更多不变量）
