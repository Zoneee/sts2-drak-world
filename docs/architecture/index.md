## 概览
本仓库围绕 STS2 弃牌触发模组组织代码、资源、构建和诊断逻辑。当前架构重点不是抽象出通用平台，而是保持“可扩展卡牌 + 可观察运行时 + 可验证部署链路”的最小闭环。

## 核心规则
- 入口 `Main.cs` 只保留初始化和补丁装配职责。
- 卡牌注册优先通过声明式标注完成，不在入口维护手工注册表。
- 运行时诊断优先放在 `ModelDb.Init()` 后，避免提前注入导致重复模型问题。
- 资源部署必须同时考虑 DLL、manifest、`.pck` 与 Godot 导入缓存。
